Shelf.nu is an open-source IT asset and equipment management platform built for teams who need to know what they have, where it is, and who's using it. Whether you're managing a few dozen devices or thousands of assets across multiple locations, Shelf gives you QR-tagged assets, calendar-based bookings, custody tracking, and role-based access control in a single product.

## What Shelf does

Shelf tracks the full lifecycle of physical assets — from procurement to retirement. Every asset gets a QR code that team members can scan to check it out, update its location, or view its history. Bookings let you reserve equipment in advance and prevent double-bookings. Custody tracking ensures accountability: you always know who has what.

## Get up and running

<Steps>
  <Step title="Create a Supabase project">
    Set up a free Supabase project for your database, authentication, and file storage. Follow the [Supabase setup guide](/supabase-setup) for step-by-step instructions including OTP configuration and storage buckets.
  </Step>
  <Step title="Clone and configure">
    Clone the repository, copy `.env.example` to `.env`, and fill in your Supabase credentials. The [local development guide](/local-development) covers SSL setup and all required environment variables.
  </Step>
  <Step title="Initialize the database">
    Run `pnpm webapp:setup` to generate the Prisma client and apply all database migrations against your Supabase project.
  </Step>
  <Step title="Start the app">
    Run `pnpm webapp:dev` and open `https://localhost:3000`. Create your first workspace and start adding assets.
  </Step>
</Steps>

## Deploy to production

Shelf deploys to [Fly.io](/deployment/fly-io) with GitHub Actions for automated CI/CD, or you can run it anywhere with [Docker](/deployment/docker).
